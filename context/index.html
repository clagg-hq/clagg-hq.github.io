Context

Standards, style, and reference

The point of a context layer is simple: keep preferences, standards, and useful operating material close enough to use.

Style

Writing defaults

  • plain language
  • active voice
  • low fluff
  • specific over abstract
  • shorter when in doubt

Standards

What good looks like

  • recommendations should be usable
  • drafts should sound like Chris Trag, not a model
  • plans should end in explicit next actions
  • repo and note hygiene matter
